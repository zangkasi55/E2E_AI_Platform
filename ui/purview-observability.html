Microsoft Purview — governance observability

The design-time / data-governance observability plane. It answers "is this source approved, what PII does it carry, and where did this number in the memo come from?" — the complement to the runtime / execution plane in the token monitor. The PoC reproduces the information shape of four real Purview surfaces (Data Map lineage, Classification insights, Scan results, Data Estate Insights) so the capability reads faithfully from file:// with no Azure.
